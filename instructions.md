# Coding Instructions

Prefer minimal targeted edits.

Do not rewrite entire files unless requested.

When relevant code is already visible, analyze it directly.

Do not repeatedly request nearby line ranges.

Do not ask for the same context twice.

Prefer best-effort debugging over repeated context gathering.

Keep responses concise and implementation-focused.

Preserve existing code style unless inconsistent.

Avoid speculative changes outside the requested scope.

When making edits:
- explain the root cause first
- then provide the fix
- avoid speculative changes
